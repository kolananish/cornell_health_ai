# VocalHealth AI — PHQ-9 Depression Screener v3

**Production model for passive voice-based depression risk screening.**
Trained on Bridge2AI-Voice Adult dataset (PhysioNet v3.0.0).

---

## What This Model Does

Given two short voice recordings and basic patient metadata, the model outputs a
calibrated probability that the patient scores PHQ-9 ≥ 10 (moderate or worse
depression). A boolean flag is generated by comparing the probability to a
pre-validated operating threshold.

This is a **decision-support tool for clinicians** — not a diagnostic device.

---

## Model Architecture

```
Voice clip 1 (rainbow passage)  ─┐
Voice clip 2 (free speech)       ├─> acoustic feature extraction (librosa)
                                 │   → 55 features per clip → median across clips
                                 │
Demographics (age, sex, edu)     ┤
Psych history flags (9 binary)   ┘
                                 │
                   ┌─────────────▼──────────────┐
                   │     ColumnTransformer        │
                   │  acoustic → StandardScaler   │
                   │  demo     → StandardScaler   │
                   │  psych    → passthrough (0/1)│
                   └──────────┬─────────────┬────┘
                              │             │
                   ┌──────────▼──┐   ┌──────▼──────────┐
                   │  Huber      │   │  TabPFN          │
                   │  Regressor  │   │  Regressor       │
                   │  (ordinal   │   │  (ordinal        │
                   │  PHQ total) │   │  PHQ total)      │
                   └──────┬──────┘   └──────┬───────────┘
                          │                 │
                   min-max normalise each score → average
                          │
                   ┌──────▼───────────────────┐
                   │  IsotonicRegression       │
                   │  (OOF-fitted calibration) │
                   └──────┬───────────────────┘
                          │
                   P(PHQ-9 >= 10) ∈ [0, 1]
```

**Key design choices:**
- **Ordinal target**: models train on continuous PHQ-9 total (0–27), not a binary
  label. This preserves near-threshold gradient (PHQ=8 vs PHQ=2 differ; binary
  lumps them together). Universally improved AUC by +0.019–0.054 across all model
  families tested.
- **Ensemble**: Huber (robust linear) + TabPFN (in-context learning). Both
  trained on the ordinal target; their scores are min-max normalised before
  averaging to handle different raw scales (~0–27 vs ~0–1).
- **Isotonic calibration**: OOF-fitted piecewise-monotone regression maps raw
  ensemble scores to valid probabilities without distorting rank order.
- **ColumnTransformer**: psych binary flags are passed through without scaling;
  scaling binary indicators distorts their meaning.

## Inputs

### Voice Recordings
Two audio clips, each ~30–45 seconds:

| Task | Description |
|---|---|
| **Rainbow Passage** | Patient reads a fixed phonetically balanced text (covers all English phonemes). Standardised; controls for content variation. |
| **Free Speech** | Patient speaks freely about their mood, energy, and sleep over the past week. Captures spontaneous prosody and emotional expression. |

Both clips are processed identically. The 55 acoustic features are extracted from
each clip independently, then **median-aggregated** across clips (same aggregation
used during training).

### Acoustic Features (55)

**Prosody (18)**
| Feature | Clinical Meaning |
|---|---|
| `speaking_rate` | Syllables/sec — psychomotor retardation slows speech |
| `articulation_rate` | Speaking rate / phonation ratio |
| `phonation_ratio` | Fraction of time voicing |
| `pause_rate` | Pauses per second |
| `mean_pause_duration` | Longer pauses → cognitive slowing |
| `mean_f0_hertz`, `std_f0_hertz` | Fundamental frequency in Hz |
| `F0semitoneFrom27.5Hz_sma3nz_amean/stddevNorm/pctlrange0-2` | Pitch in semitones; narrowed range = flat affect |
| `loudness_sma3_amean/stddevNorm/pctlrange0-2` | Amplitude mean, variability, range |
| `mean_intensity_db`, `std_intensity_db` | Intensity statistics |
| `VoicedSegmentsPerSec` | Voice onset rate |
| `MeanVoicedSegmentLengthSec` | Duration of voiced runs |
| `MeanUnvoicedSegmentLength` | Duration of pauses |

**Voice Quality (11)**
| Feature | Clinical Meaning |
|---|---|
| `mean_hnr_db`, `std_hnr_db` | Harmonics-to-noise ratio; lower = breathier voice |
| `local_jitter`, `rap_jitter` | Period-to-period pitch instability |
| `local_shimmer`, `localDB_shimmer` | Amplitude instability |
| `cepstral_peak_prominence_mean/std` | Overall voice quality index |
| `jitterLocal_sma3nz_amean` | OpenSMILE-compatible jitter |
| `shimmerLocaldB_sma3nz_amean` | OpenSMILE-compatible shimmer |
| `HNRdBACF_sma3nz_amean` | OpenSMILE-compatible HNR |

**Spectral (17)**
| Feature | Clinical Meaning |
|---|---|
| `mfcc1–4_sma3_amean` (4) | Spectral envelope shape; MFCC-1 correlates with spectral tilt |
| `mfcc1–4_sma3_stddevNorm` (4) | MFCC variability over time |
| `spectral_skewness`, `spectral_kurtosis` | High-frequency energy distribution |
| `spectral_slope`, `spectral_tilt` | Overall spectral roll-off |
| `spectral_gravity` | Spectral center of mass |
| `alphaRatioV_sma3nz_amean` | Low/high band energy ratio |
| `hammarbergIndexV_sma3nz_amean` | 0–2 kHz vs 2–5 kHz dominance |
| `slopeV0-500_sma3nz_amean` | Low-band spectral slope |
| `slopeUV500-1500_sma3nz_amean` | Mid-band spectral slope |

**Formants (9)**
| Feature | Clinical Meaning |
|---|---|
| `mean_f1_loc`, `std_f1_loc` | First formant frequency and variability |
| `mean_f2_loc`, `std_f2_loc` | Second formant frequency and variability |
| `F1frequency_sma3nz_amean` | OpenSMILE-compatible F1 |
| `F2frequency_sma3nz_amean` | OpenSMILE-compatible F2 |
| `F3frequency_sma3nz_amean` | Third formant frequency |
| `F1amplitudeLogRelF0_sma3nz_amean` | F1 amplitude relative to F0 |
| `F2amplitudeLogRelF0_sma3nz_amean` | F2 amplitude relative to F0 |

### Patient Metadata (12)

**Demographics (3)** — scaled with StandardScaler
| Feature | Values |
|---|---|
| `age_num` | Continuous (years) |
| `is_female` | 1.0 = Female, 0.0 = Male, NaN = not specified |
| `edu_num` | 1 = < high school, 2 = HS/GED, 3 = some college / associate's, 4 = bachelor's, 5 = graduate |

**Self-Reported Psych History (9)** — binary 0/1 from intake form; NOT scaled
| Feature | Condition |
|---|---|
| `sr_depression` | Depression |
| `sr_gad` | Generalized Anxiety Disorder |
| `sr_ptsd` | PTSD |
| `sr_insomnia` | Insomnia |
| `sr_bipolar` | Bipolar Disorder |
| `sr_panic` | Panic Disorder |
| `sr_soc_anx_dis` | Social Anxiety Disorder |
| `sr_ocd` | OCD |
| `any_psych_sr` | 1 if any of the above = 1 |

> These flags come from the patient's existing EHR or intake questionnaire — they
> represent history, not current diagnosis. They are the strongest single
> predictors in the model (SHAP analysis).

## Performance

All metrics are **5-fold out-of-fold (OOF)** — no data leakage.
Evaluated on N=754 participants, Bridge2AI-Voice Adult v3.0.0.
Positive class: PHQ-9 ≥ 10 (n=104, 13.8% prevalence).

### Model Comparison

| Model | AUC | Notes |
|---|---|---|
| v2 baseline (LR L1, binary target) | 0.734 | Previous production model |
| Huber ordinal solo (raw) | 0.759 | +0.025 vs v2 |
| Huber ordinal solo (calibrated) | 0.740 | +0.006 vs v2 |
| **Huber + TabPFN ensemble (raw)** | **0.761** | Best raw discriminability |
| **Huber + TabPFN ensemble (calibrated)** | **0.744** | **Production model** |

### Confusion Matrix — Balanced Threshold (Youden-J = 0.091)

```
                    Predicted Negative   Predicted Positive
Actual Negative         TN = 427              FP = 223
Actual Positive         FN = 25               TP = 79
```

| Metric | Value | Interpretation |
|---|---|---|
| **Sensitivity (Recall)** | **0.760** | 76% of true cases flagged |
| **Specificity** | **0.657** | 66% of healthy patients correctly cleared |
| **PPV (Precision)** | **0.262** | 1 in 4 flagged patients has PHQ-9 ≥ 10 |
| **NPV** | **0.945** | 94.5% — strong negative predictive value |
| **F1 Score** | **0.389** | Harmonic mean (imbalanced class) |

### Why AUC — Not Accuracy

At 13.8% prevalence, a model that predicts "no depression" for everyone achieves
**86.2% accuracy** — but catches zero cases. AUC measures discrimination across all
thresholds regardless of prevalence, making it the correct metric for imbalanced
clinical screening tasks.

## How to Reproduce Training

### Prerequisites

```bash
# Dataset must be in adult/ (Bridge2AI-Voice v3.0.0, PhysioNet)
# Python environment with dependencies:
pip install scikit-learn tabpfn polars pandas numpy scipy librosa joblib
```

### Run

```bash
python train_psych_screener_v3.py
```

This will:
1. Load `adult/features/static_features.tsv` + `adult/phenotype/questionnaire/phq9.tsv`
2. Load demographics and self-reported psych history
3. Filter to `rainbow-passage` and `free-speech-3` tasks; median-aggregate features
4. Run baseline comparison (v2 LR vs ordinal approaches)
5. Run nested 5×3-fold GridSearchCV for Huber hyperparameters (Spearman scorer)
6. Fit isotonic calibration OOF
7. Train final Huber+TabPFN ensemble on full data
8. Save `models/psych_screener_v3.joblib` and `models/psych_screener_v3_metadata.json`

Training time: ~5–10 minutes on CPU (TabPFN OOF is the bottleneck).

### Required Files in `psych_models.py`

`psych_models.py` must be importable before loading the joblib bundle:

```python
from psych_models import CalibratedOrdinalClassifier, OrdinalEnsembleClassifier
import joblib
bundle = joblib.load("psych_screener_v3.joblib")
```

## Why These Modelling Choices

**Why ordinal regression instead of binary classification?**
PHQ=9 and PHQ=2 are both labelled 0 in binary training, but they sit very
differently relative to the clinical threshold. Training on the continuous 0–27
score lets the model learn this gradient; the binary label is only used for
calibration and evaluation.

**Why Huber loss?**
PHQ-9 distributions are right-skewed with a few extreme scores (PHQ=22–27,
n<10). Mean squared error would over-weight these. Huber is quadratic near zero
and linear in the tails — less sensitive to PHQ=22 outliers pulling the regression
line away from the bulk of the data.

**Why TabPFN as ensemble partner?**
TabPFN is a meta-learned prior-data fitted network — it does in-context learning
over tabular data without gradient updates. It adds a complementary inductive bias
to the linear Huber model. At N=754, tree methods (LightGBM, XGBoost) underfit
compared to regularised linear models; TabPFN's meta-learned prior closes this gap.

**Why not deep MLP or fine-tuned speech model?**
- MLP with focal loss: AUC=0.668 — insufficient depth for N=754 tabular data
- Raw audio models (wav2vec2, HuBERT): Bridge2AI only provides pre-extracted
  features, not raw waveforms, in the public release

**Why these two voice tasks?**
Ablation study (experiment_voice_psych_screening.py) showed:
- Rainbow passage has the strongest standalone signal (AUC=0.570)
- Free speech adds complementary signal (combo: AUC=0.613 voice-only)
- Adding caterpillar passage or prolonged vowel did not improve

## Limitations

- **Sample size**: N=754 is small for ML; performance may not generalise across
  demographics not well-represented in Bridge2AI.
- **No raw audio**: Features are pre-extracted from OpenSMILE. Real-world
  performance depends on microphone quality and recording conditions.
- **Self-reported psych history** is the strongest predictor — in populations
  without EHR intake data, voice-only AUC drops to ~0.61.
- **Not FDA-cleared**. For research and decision-support only.
- **PHQ-9 ≥ 10** is a standard screening threshold but does not constitute
  diagnosis of major depressive disorder.
